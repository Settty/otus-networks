#  IPv4/v6

###  Задание:

1. Разработаете и задокументируете адресное пространство для лабораторного стенда.
2. Настроите ip адреса на каждом активном порту
3. Настроите каждый VPC в каждом офисе в своем VLAN.
4. Настроите VLAN управления для сетевых устройств
5. Настроите сети офисов так, чтобы не возникало broadcast штормов, а использование линков было максимально оптимизировано
6. Используете ipv4 и ipv6


###  1. Адресное пространство с использованием IPv4 и IPv6.

Сеть разрабатывалась следующим образом:

Пример Триада: Каждая внутреняя подсеть Ipv4 в Триаде имеет адрессацию 10.10.10.X /30, где X номер подсети. 3 октет - цифра 10 говорит о том, что эта подсеть выделена для Триады. Сумарная подсеть 10.10.10.0/24.

Для IPv6 выбрана Unicast Global сеть 20AA:BBCC:10:0::/64 где 3 хекстет указывает что это IP для Триады (цифра 10), 4 хекстет это номер посдети совпадающий с номером подсети для IPv4 и последний хекстет это IPv6 конечного узла совпадающий с IPv4 конечного узла.

* Пример

| Equip | Port | AddrTyp | Address                  | Network                | Description     |
|-------|------|---------|--------------------------|------------------------|-----------------|
|	R26	|	e0/0	|	IPv6		|	20AA:BBCC:10:8::9/64	|	20AA:BBCC:10:8::/64	|	to	R24	e0/1	|
|	R26	|	e0/0	|	IPv6	LL	|	FE80::26	|	FE80::/10	|	to	R24	e0/1	|
|	R26	|	e0/1	|	IPv4		|	140.140.140.1	|	140.140.140.0/28	|	to	R28	e0/0	|

Для Москвы выбрана сеть 10.10.20.X /30, где X номер подсети. 3 октет - цифра 20 говорит о том, что эта подсеть выделена для Москвы. Сумарная подсеть 10.10.20.0/24.

Для Питера сеть 10.10.30.X 

IPv6 назначается аналогичным образом для Москвы и Питера

Все link-Local адреса для IPv6 имеют адрес FE80::X где X - номер маршрутизатора/коммутатора согласно схемы. Т.е. если роутер R20 то IPv6 FE80::20

### 5. Настроите сети офисов так, чтобы не возникало broadcast штормов, а использование линков было максимально оптимизировано.

* #### г.Москва (AS 1001) Distrebution-коммутаторы  SW4 и SW5 делят сеть на L2 сегмент и L3 сегмент.

* L2 сегмент 

В L2 сегменте для отказоустойчивости используется Spanning Three Protocol (RPVST+).

Так же для отказоустойчивости на Distrebution-коммутаторах SW4 и SW5 используется протокол HSRP.

Согласно задания На Access коммутаторах созданы VLAN.

VLAN 20 принадлежит коммутатору SW3. 

VLAN 30 принадлежит коммтутаору SW2.

* VLAN 20

Сеть VLAN 20 - 192.160.20.0 /24 

interface Vlan20

    ip address 192.168.20.2 255.255.255.0
    standby 1 ip 192.168.20.1
    standby 1 priority 110
    standby 1 preempt
 
Default Gateway 192.168.20.1 

(Standby HSRP) - коммутатор SW4 является ACTIVE для VLAN 20 (установлен Priority 110)

Коммутатор SW4 является Root для VLAN 20 (STP) 

    spanning-tree vlan 20 priority 24576


* VLAN 30

В данной топологии используется терминология Routed Campus Architecture т.е. маршрутизация осуществляется сразу на Access коммутаторах SW9 и SW10.

Соответсвенно в данной топологии отсутсвует STP и протоколы группы FHRP. 

Маршрутизация осуществляется на созданных SVI интерфейсах.

SW9 SVI - 192.168.40.1 /24

SW10 SVI - 192.168.50.1 /24

VLAN 50 принадлежит коммутатору SW10

    Group  Port-channel  Protocol    Ports
